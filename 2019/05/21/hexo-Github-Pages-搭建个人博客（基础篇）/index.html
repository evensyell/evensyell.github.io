Hexo + Github Pages 搭建个人博客（基础篇）

发表于 2019.5.21 | 更新于 2019.5.23
本文字数： 2.4k

写在前面：

  • 本文力求以最少的字数完成教程目的，但也会对重要的注意事项进行提示。
  • 适用人群：知道自己要做什么，只想要一篇能达成目的、不啰嗦的教程之人。

环境安装(node.js、git)

  • 运行环境：node.js —— 官网下载

  • 版本控制工具：Git —— Github镜像下载

  • 静态网页生成器：Hexo —— 在任意命令行(Cmd、PowerShell或Git Bash)中输入：

    $ cnpm install -g hexo-cli

    cnpm 是从淘宝镜像源进行包管理的指令，用以替代速度极慢的官方的 npm。后续需要运行的一切 npm 指令都可以替换为 cnpm。如果没有，可以运行以下命令安装：

    $ npm install -g cnpm --registry=https://registry.npm.taobao.org

配置Github

启用Github Pages

  1. 登陆Github。
  2. 选择右上角的 new repository 新建一个储存库。

    注意：repository name 的格式必须为 username.github.io，这也就是你博客的网址。其中 username 为你的 Github 用户名，填写错误将不能通过git推送更新博客。

  3. 打开刚刚创建的储存库，进入 setting ，往下拉找到 Github Pages 一项，确保其已经开启，或在随便选择一个 theme 后能够开启，此时你应当已经可以访问 username.github.io。

    对于复杂或陌生网页上文本信息的寻找和定位，如 “setting”，请善用浏览器的查找功能 Ctrl+F，在文档中也是如此。

关联 Git 与 Github

  1. 在任意位置右键，选择 Git Bush Here，依次输入如下命令：

    $ git config --global user.name "你的GitHub用户名"
$ git config --global user.email "你的GitHub注册邮箱"
$ ssh-keygen -t rsa -C "你的GitHub注册邮箱"

    第三行命令要求输入密码，该密码用于每次 Git 推送更新时的身份验证，默认不需要密码，直接回车到底即可。

  2. 打开电脑的 .ssh 文件夹 (C:\Users\username\.ssh)，将id_rsa.pub 密钥文件的内容全部复制。

  3. 打开 github.com/settings/keys 页面，点击 New SSH Key。title 随便填，将刚才复制的内容粘贴到 Key 一栏，Add，搞定。如此一来今后的 git 推送将使用SSH的身份验证方式，更安全，也更省心。

初尝 Hexo

  1. 在合适位置创建 blog 文件夹，今后博客相关的所有的文件都放这个文件夹。

  2. 启用hexo。在 blog 文件夹内右键打开 Git Bash(后面将省略类似说明)，输入：

    $ hexo init # 在当前目录创建hexo的实例

    命令执行的结果一般总有反馈，执行成功或是失败、失败的原因或建议的处理方式，请善用这些信息。

    主动理解与思考 >> 百度一下 >> 询问他人。

  3. 检查hexo。输入：

    $ hexo g
# 生成静态网页

$ hexo s
# 用本地服务器进行网页预览
# 在浏览器输入 localhost:4000 就可以看见hexo生成的网页了

  4. 关联 Hexo 与 Github 储存库
    打开 blog/_config.yml 文件，拉到最下面，将最后一段修改为如下所示。

    deploy:
    type: git
    repo: https://github.com/username/username.github.io.git
    branch: master

    别忘了替换 username；每个冒号后面一定要有一个英文空格

    今后还会频繁、大量的编辑这种配置文件，一款开启代码高亮的编辑器将能够帮助你节省很多精力和时间。推荐微软的开源编辑器 VS Code，它免费、轻量、响应快、插件丰富（我甚至用它来写了这篇Markdown教程）

  5. 常用的hexo指令。

    # 查看所有hexo指令和功能
$ hexo help

# 根据当前配置文件生成网页，所有内容存入 blog/public 文件夹内
$ hexo g

# 建立本地服务器，使可通过 localhost:4000 访问 public 文件夹中的内容，即本地预览已生成的网页
# 在命令行内按 Ctrl+C 可停止本地预览
$ hexo s

# 推送public内的网页到云端服务器
$ hexo d

# 依据当前配置，生成网页并推送
$ hexo g -d

# 新建名为 "art_name" 的新文章
$ hexo new "art_name"

# 清除缓存，即清空 public 文件夹内容
$ hexo clean

    新建的文章为.md格式(Markdown)，存放于 blog/source/_posts/ 。

    通过hexo new 命令新建的.md文件自带了一个 Front-matter 参数段，如下所示。

    Hexo 通过读取该参数段来获得文章的时间、标题、标签等信息。你也可以在其他位置新建.md文件，编辑好内容后放入 _posts 文件夹，只是别忘了手动添加这些参数段：

title: Hexo + Github Pages 搭建个人博客（基础篇）
