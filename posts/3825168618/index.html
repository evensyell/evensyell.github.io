文章链接唯一化

  • 很可能你会数次更改文章题目或者变更文章发布时间，在默认设置下，文章链接都会随之发生改变，不利于搜索引擎收录，也不利于分享。将文章链接唯一化、永久化才是长久之道。这里选用对初始标题+时间进行md5，然后再转base64，保存在front-matter中作为链接的方案。

    实现

    1. 安装插件：

      cnpm install hexo-abbrlink –save

  1. 修改站点配置文件的 permalink 值（blog/_config.yml）

    permalink: posts/:abbrlink/

    注意键值对的冒号后一定要有一个英文空格。

  2. 在文件最后添加上abblink配置：

    # abbrlink config
abbrlink:
alg: crc32
rep: dec

效果

  1. 预览或推送后，文章文档的 Front-matter 参数块中出现了 abbrlink: 3825168618 这样的字段。
  2. 网页链接变为 http://localhost:4000/posts/3818330983/，今后无论怎么修改文章题目或内容，只要不去手动修改 abbrlink 值，链接就将永久不变。

首页显示文章摘要

  • 如果你忽视了基础篇中的一个小提示，你可能会发现，尽管首页显示的是一个所有文章的列表，但每一篇文章都是全文显示，这样浏览起来可不舒服。更普遍的习惯是启用预览摘要。

    实现

    方法1 手动插入标签（推荐）

  • 在文章编辑时，先编写摘要部分，接着空一行输入 <!--more--> ，然后再换行开始编写正文。标签之前的部分将作为摘要显示在首页，之后的部分将作为正文在点击 阅读更多>> 按钮后展示。

    在标签前一空行插入 --- ，即分隔符，会让摘要更为美观。

    方法2 设置自动摘要

  • 在主题配置文件中进行如下设置：

    auto_excerpt:

    enable: true    #启用自动摘要
length: 100     #设置摘要字数

使用 Github 图床

  • 现在已经有很多免费/收费图床和方便传图的小工具可选，其便利性远超使用本地图片手动管理，值得每个人都尝试一下。但是，考虑到国内的网络特色，免费的图床往往速度慢，高速的图床往往要求备案，尝试过免费又高速的七牛云，被诱导实名认证开始用之后才告诉你，上传的图片一个月后会自动删除……除非绑定上备过案的域名，这实在是折腾。于是在此推荐简单省事的 Github 图床。

    实现

  1. 新建 Github 仓库，例如 img

  2. 访问 Github Devloper Setting ，点击右上角 Generate New Token，随便命名，勾选repo权限，拉到最下面，点击生成Token，保存备用（这玩意只出现这一次，丢失只能删除后重建）

  3. 安装开源图床管理工具 Picgo ，进入图床设置界面填写如下：

    仓库名填写格式：账户名/仓库名（我的仓库名是img_bed）；
    分支名：统一填写“master”；

    存储的路径可不填，如果填写blog/，就会在repository下创建一个“blog”文件夹存放图片，否则所有图片都在根目录下；

    自定义域名也可不填，在上传图片后成功后，PicGo会将“自定义域名+上传的图片名”生成的访问链接，放到剪切板上，填写格式必须为：https://raw.githubusercontent.com/用户名/RepositoryName/分支名。

    4.配置完成后，开始使用吧，使用方法十分简单，复制图片、上传图片、复制Markdown代码、粘贴到文章中。请多加思考，必要时查阅 Picgo 项目主页

效果

  • 本博客所有图片均使用 Github 图床。
